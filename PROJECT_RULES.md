# PROJECT_RULES.md

## プロジェクト概要

### サイト名
旅するマネーノート

### コンセプト
海外旅行のお金に関する情報を発信するSEOメディア。

比較サイトではなく、「旅行前に知っておきたいお金の知識」を分かりやすく届けることを目的とする。

---

# プロジェクトの目的

このサイトは検索ユーザーの疑問を解決するための情報サイトである。

目的は以下の3つ。

1. 海外旅行前のお金に関する悩みを解決する
2. Google検索で長期的に評価される記事を蓄積する
3. 読者が安心して旅行準備できるメディアを育てる

---

# ターゲット

- 20〜40代
- 海外旅行初心者
- 女性ユーザーを中心
- 外貨両替について調べている人
- eSIM・クレジットカード・旅行準備を調べている人

---

# サイトの世界観

旅行好きの女性が運営している旅行メディア。

企業サイトではなく、

「旅行前の自分が読みたかったサイト」

を目指す。

安心感・読みやすさ・親しみやすさを重視する。

---

# デザインルール

- スマホファースト
- レスポンシブ対応
- シンプルで読みやすい
- 余白を十分に使う
- カード型デザイン
- デザインは統一する
- 勝手に大きく変更しない

---

# HTMLルール

HTML5で作成する。

必ず以下を使用する。

- header
- nav
- main
- article
- section
- aside（必要な場合）
- footer

H1はページ内に1つだけ。

見出し構造を崩さない。

---

# SEOルール

全ページに以下を設定する。

- title
- meta description
- canonical
- OGP
- Twitter Card

構造化データを利用できる場合は実装する。

画像には必ずalt属性を設定する。

パンくずリストを設置する。

関連記事を設置する。

内部リンクを積極的に追加する。

---

# CSSルール

既存CSSを優先する。

CSSを重複させない。

共通コンポーネントを使用する。

---

# JavaScriptルール

必要最低限のみ使用する。

ページ表示速度を最優先する。

---

# 記事ページの構成

記事には必ず以下を設置する。

- アイキャッチ
- 更新日
- 目次
- 本文
- FAQ
- まとめ
- 関連記事
- CTA

---

# 記事作成ルール

初心者にも理解できる文章で書く。

結論を最初に書く。

比較表を積極的に使う。

公式情報を優先する。

推測だけでは書かない。

一次情報を優先する。

# 画像ルール

Canvaで作成する。

画像は軽量化する。

画像ファイル名は内容が分かる英単語を使用する。

例

thailand-money-exchange.webp

---

# URLルール

英語スラッグを使用する。

例

/thailand/money-exchange/

/korea/esim/

/travel-preparation/

---

# 更新ルール

新しい記事を追加した場合は

- カテゴリページ
- 関連記事
- パンくずリスト
- サイト内導線
- 内部リンク
- トップページ（必要な場合のみ）

を更新する。

内部リンクは最低3件追加する。

同じカテゴリの記事同士を優先してリンクする。

---

# 開発ルール

WordPressは使用しない。

HTML・CSS・JavaScriptのみで構築する。

GitHubで管理する。

GitHub Pagesで公開する。

既存デザインを維持する。

変更は必要最小限とする。

既存コードを壊さない。

勝手にCSSを書き直さない。

勝手にレイアウトを変更しない。

改善内容は必ず既存デザインを尊重する。

新しいページ追加時は既存テンプレートを流用する。
